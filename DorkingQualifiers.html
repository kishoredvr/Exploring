🌐 Universal Search Operators

These core operators work across most major search engines – Google, Bing, DuckDuckGo, and Yandex. They form the backbone of any dorking query and should be mastered first.

OperatorDescriptionExample
" "Search for an exact phrase or sequence of words."index of /admin"
-Exclude a term from the results (minus sign).apache -tomcat
OR / |Logical OR; find results containing at least one of the terms.php OR asp OR jsp
ANDLogical AND (often the default behaviour, but can be used explicitly).admin AND login
*Wildcard; matches any character or whole word within a phrase."database * password"
( )Grouping for complex boolean logic.(php OR asp) login
+Force inclusion of a stop word or common term.+the +who
~Include synonyms (tilde; may not work on all engines).~mobile phone

🔍 Google Search Operators

Google offers the largest collection of advanced operators. Many are essential for passive reconnaissance and targeted dorking. All universal operators above are also supported.

🔎 Bing Search Operators

Bing supports most universal operators and offers several unique ones ideal for OSINT and detailed web reconnaissance.

OperatorDescriptionExample
site:Limit results to a specific domain or TLD.site:microsoft.com
domain:Restrict to a root domain (more strict than site:).domain:example.org
url:Pages with a specific word in the full URL.url:download
title:Find word in the page title (Bing's intitle).title:security
body:Find word in the body text (Bing's intext).body:"confidential"
filetype:Filter by file extension.filetype:xlsx budget
language:Results in a specific language.language:fr
loc: / location:Geographic region (country/region code).loc:GB
feed:Find RSS/Atom feeds related to a query.feed:cybersecurity
hasfeed:Pages that have an RSS feed.site:example.com hasfeed:
ip:Search for websites hosted on a specific IP address.ip:192.0.2.1
prefer:Boost results matching a keyword (preference).prefer:tutorial

contains:Only results that contain a specific file type link.contains:pdf
norelax:Disable automatic relaxation of search constraints.norelax:site:example.com admin
noalter:Prevent Bing from altering the query.noalter:site:example.com
literalmeta:Force literal interpretation of meta keywords.literalmeta:site:example.com

🦆 DuckDuckGo Operators

DuckDuckGo prioritizes privacy and supports a streamlined set of operators. It lacks some Google-specific ones (e.g., link:) but covers most core dorking needs.

OperatorDescriptionExample
site:Search within a specific site or TLD.site:github.com
-site:Exclude results from a specific site.cats -site:example.com
intitle:Word in the page title.intitle:login
inurl:Word in the page URL.inurl:admin
filetype:Filter by file extension (works like Google).filetype:pdf manual
ext:Same as filetype: ; search by extension.ext:sql backup
~" "Experimental: find semantically similar phrases.~"cats and dogs"
region:Boost results from a specific region (e.g., us-en, uk-en).region:de
!Bang: search directly on another site (e.g., !amazon).!w ethical hacking
\Go directly to the first search result (I'm feeling lucky).\futurama
💡 Note: DuckDuckGo also supports !safeon and !safeoff to toggle SafeSearch, and timelimit: parameters (d, w, m, y) when used programmatically.

🇷🇺 Yandex Search Operators

Yandex provides a robust set of operators for deep searching, especially valuable for Russian and CIS web assets. Many operators mirror Google's but with additional region-specific capabilities.

OperatorDescriptionExample
site:Restrict search to a domain and its subdomains.site:*.ru
domain:Strict domain match (usually without subdomains).domain:yandex.ru
host:Similar to site:, matches hostname.host:mail.example.com
rhost:Match hostname in reverse order (TLD first).rhost:com.seosly.www
url:Find pages with a specific URL (use quotes for special chars).url:seosly.com
mime:File type by MIME (e.g., pdf, xls, ppt).mime:pdf
lang:Language of the document (ISO code). Supports ru, uk, be, en, fr, de, kk, tt, tr.lang:en
date:Filter by date (YYYYMMDD or * for day/month).date:20240601
title:Word in the title (like intitle).title:"index of"
inurl:Word in the URL.inurl:admin
! <word>Match the exact form of a word (case, number, tense).buy !apples wholesale
+ <word>Match stop words (auxiliary parts of speech, pronouns).work +from home
[ ]Matches the order of words (any grammatical forms / stop words inside).tickets [london paris]
$ anchor()Search for pages linking with specific anchor text.$ anchor(seo)
( ) and |Group words in complex queries.buy (coat|jacket)
region:Geographical region ID (e.g., 213=Moscow).region:213
💡 Yandex-Specific Notes: The !, +, [ ], and $ anchor() operators are unique to Yandex and are extremely powerful for precise language-based searches. Yandex's lang: operator supports Russian, Ukrainian, Belarusian, English, French, German, Kazakh, Tatar, and Turkish.

🐙 GitHub Code Search Qualifiers

GitHub's advanced code search can uncover exposed secrets, configuration files, and proprietary code. These qualifiers work in the main search bar and are distinct from non-code search (issues, users).

QualifierDescriptionExample
language:Programming language.language:python
path:File path contains a string (supports glob and regex).path:/config/
filename:Exact file name.filename:.env
extension:File extension.extension:pem
org:Repository belonging to an organization.org:google
repo:Specific repository (user/repo).repo:torvalds/linux
user:Code owned by a user.user:johndoe
enterprise:Files within an enterprise account.enterprise:octocorp
size:File size in bytes (with >, <, range).size:>5000
is:State of the item (archived, fork, vendored, generated).is:archived
content:Restrict search to file content only (not paths).content:README.md
symbol:Search for symbol definitions (functions, classes).language:go symbol:WithContext
license:Filter by license type (e.g., MIT, Apache-2.0).license:MIT
NOTExclude keyword from code results.NOT test
AND / ORBoolean logic (AND is default when terms separated by space).sparse OR index
💡 Pro Tips: Combine qualifiers with exact phrases like "BEGIN RSA PRIVATE KEY" to locate accidentally committed private keys. Use path:*.txt for glob patterns or path:/(^|\/)README\.md$/ for regex. GitHub code search is case-insensitive by default; use regex /(?i)pattern/ for case-sensitive matching.
